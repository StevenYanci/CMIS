# CMIS — 演唱会文创智能策划系统

## 核心理念

> 将艺人的情绪价值、演唱会体验、城市文化与粉丝消费心理，转化为可规模化销售的情绪消费产品系统。

本系统不是"给艺人做周边"，而是一套完整的**商业化情绪产品决策引擎**。

---

## 标准输入结构

用户只需提供以下信息，其余全部自动执行：

```yaml
artist_name:      # 艺人/组合名称
concert_name:     # 演唱会名称（可选）
concert_date:     # 演唱会日期
concert_city:     # 举办城市
concert_venue:    # 场馆名称
budget_range:     # 文创预算范围（可选，影响SKU结构）
target_sku_count: # 期望SKU数量（可选，默认8-12个）
```

---

## 执行流程总览

收到输入后，**严格按以下6层顺序执行**，每层完成后输出小结，全部完成后汇总为《全案策划书》。

```
Layer 1 → 数据采集与艺人分析
Layer 2 → 演唱会情绪系统分析  
Layer 3 → 城市/场馆/IP权重分析
Layer 4 → 文创趋势与消费洞察
Layer 5 → 商品决策与爆款预测
Layer 6 → 创意生成与运营方案
```

详细执行规范见 `references/layers.md`。

---

## 最终输出结构

完成所有分析后，输出标准格式的《演唱会文创全案策划书》，包含以下章节：

| # | 章节 | 核心内容 |
|---|------|---------|
| 1 | 艺人IP分析 | 国民度、粉丝画像、审美偏好、情绪热点 |
| 2 | 演唱会情绪地图 | 主题解析、情绪曲线、视觉元素提取 |
| 3 | 城市与场馆分析 | 消费能力、地域文化、场馆IP权重 |
| 4 | 文创趋势报告 | 当下爆款品类、材质风格、消费趋势 |
| 5 | SKU战略矩阵 | 引流/利润/品牌/稀缺款完整结构 |
| 6 | 爆款预测排行 | 每个SKU的爆款概率评分 |
| 7 | 设计关键词 | 视觉/情绪/材质/禁忌关键词 |
| 8 | 包装与文案方向 | 开箱体验、仪式感、社交传播点 |
| 9 | 定价策略 | 各SKU定价区间与价格梯度 |
| 10 | 销售运营方案 | 预售机制、现场布局、长尾销售 |
| 11 | 风险提示 | 库存风险、设计风险、粉丝预期风险 |
| 12 | 样图提示词 | 每款SKU的AI生图提示词（Midjourney/Flux格式） |

## 关键判断原则

在执行过程中，始终遵守以下原则：

1. **情绪优先于功能**：演唱会文创的核心是情绪消费，不是实用消费
2. **审美准确是生死线**：审美错误直接导致销量失败，宁可保守也不冒险
3. **场馆元素要克制**：除非场馆本身有极高IP价值，否则场馆元素应作为隐性细节而非主视觉
4. **城市适配不可忽略**：北京、上海、成都、广深的消费偏好差异显著
5. **当下热点必须结合**：SKU必须结合艺人当前情绪热点，而非历史标签

## Web Search 指引

执行过程中，**必须使用 web search** 获取以下实时信息：

- 艺人最近热搜、争议、爆火内容（微博/小红书/抖音）
- 演唱会官宣信息与主视觉
- 城市消费数据与文娱消费习惯
- 当前文创行业爆款趋势
- 粉丝社群讨论与审美偏好

不可仅依赖训练数据，因为**"当下情绪"是周边成败的核心变量**。
