# CMIS — 鸟巢限定款品类发现框架 v5.0

> ## 核心定位（v5.0重要更新）
>
> 本文件服务于「鸟巢×艺人联名限定款」这一特定产品类型的品类发现，不是通用演唱会周边规划。
>
> **鸟巢限定款的产品选择有两个额外约束（v5.0新增）：**
>
> 约束1：必须能够回答「为什么这款产品只能是鸟巢限定款，而不是艺人自有产品？」
>        ——如果艺人官方店随时可以卖，那它就不是限定款，没有稀缺溢价。
>
> 约束2：必须考虑艺人自有文创对本款产品的分流率
>        ——分流率越高，预期销量越低，生产数量需相应收缩。
>
> **本文件是「发现框架」，不是「选择清单」。**
> 每次策划通过三维框架主动发现最适合的产品方向。

## 一、鸟巢限定款品类发现三维框架

```
        艺人历史爆款（已验证粉丝偏好）
               ↕
    文创市场最新趋势 ←→ 鸟巢场馆物理特性
     （避免俗套/发现新品类）   （夜间/9万人/户外）
```

在三维框架基础上，**增加「鸟巢限定款独占性验证」第四维度**：

```
三维交叉候选方向
       ↓
第四维度：独占性筛选
       ↓
只保留「艺人自有产品无法做或做不好」的方向
```

| 历史爆款 | 爆火底层逻辑 | 本次延伸方向 | 是否艺人自有产品已覆盖 |
|---------|-----------|-----------|---------------------|

### 维度C：鸟巢场馆专属产品机会

| 鸟巢特性 | 潜在产品机会 | 独占性：艺人自有产品能否做 |
|---------|-----------|------------------------|
| 夜间+9万人 = 大规模发光视觉 | [方向] | 否（鸟巢联名应援棒需官方合作）|
| 户外夏季 = 防晒/遮阳需求 | [方向] | 可能有分流 |
| 超大坐席 = 方位/编号概念 | 「第X区X排」个性化产品 | 否（艺人自有店无法提供鸟巢座位坐标） |
| 鸟巢×艺人「联名」背书 | 带鸟巢官方LOGO的联名产品 | 否（这是鸟巢限定款独有权益） |

### 第四维度：独占性筛选

对上述三维框架产生的候选方向，逐一回答：

**「如果这款产品放到艺人官方店里卖，粉丝还会来鸟巢限定款专区买吗？」**

- 答案是「不会」→ 这是真正的鸟巢限定款机会（独占性强）
- 答案是「可能会」→ 需要强化鸟巢专属叙事（独占性中）
- 答案是「一样的」→ 这款产品做了也是在跟艺人自有产品竞争（独占性弱，慎做）

## 二、产品叙事完整性要求

每款SKU必须讲清楚叙事三要素：

1. **它从哪里来**：与鸟巢×演唱会的具体联结（不是泛泛纪念品）
2. **它发生了什么**：粉丝拥有它后会做什么（用它？拍它？集它？）
3. **它去哪里**：演出结束后5年，这件产品对拥有它的人意味着什么

**额外要求（鸟巢限定款专属）：**
4. **演出结束后，粉丝为什么还珍视它？**
   ——限定款的长期价值在于「历史见证物」属性，这个价值必须在产品设计和文案中被激活

## 三、供应商定量决策框架

> ⚠️ 这是v5.0区别于所有通用文创策划系统的核心模块。
> 专为鸟巢限定款供应商设计——演出结束即下架，过量生产等于直接亏损。

### 定量决策四步法

```
Step 1: 确定净市场容量
        = 有效观演人次 × 文创转化率 × 热度系数 × (1-分流率) × 北京系数

Step 2: 对比历史参照
        找同类艺人/同规模演唱会的实际文创销售数据做交叉验证
        如果Step 1结果与历史参照偏差>30%，需重新检查假设

Step 3: 确定风险承受度
        保守：净市场容量×55%（首次合作/新艺人/热度数据缺乏）
        中值：净市场容量×75%（有历史数据/热度确定性高）
        乐观：净市场容量×90%（预售数据充分支撑/历史爆款验证）

Step 4: 分批生产安排
        第一批：选定档位的60–70%（演出前30天启动）
        启动预售收集数据
        第二批：根据预售数据在演出前10天决定是否追加
```

### 各象限产品定量建议比例

| 象限 | 在总产量中的占比 | 建议生产逻辑 | 预售比例 |
|-----|--------------|-----------|---------|
| 引流款（¥35–79） | 25–35% | 单价低，可适当多备，现场冲动消费率高 | 0%（不做预售）|
| 利润款（¥99–299） | 35–45% | 主力贡献，需要最精准的定量 | 30–40% |
| 品牌款（¥199–399） | 15–20% | 中高端，先预售再定产 | 50–60% |
| 稀缺款（¥299–599） | 5–15% | 严格限量，宁少勿多 | ≥70% |

### 预售机制：规避库存风险的最有效工具

**为什么预售比首批多生产更安全：**
- 预售订单是「确定性需求」，首批多生产是「预测性需求」
- 预售开票后5–7天的数据，反映真实需求远比任何模型更准确
- 稀缺款预售≥70%意味着最多只有30%面临库存风险

**预售设计要点：**
- 限量稀缺款必须走预售，总量=预售量/70%（预留30%现场销售）
- 预售页面显示「剩余XX套」实时倒计时，强化稀缺感
- 预售数据每天更新到定量决策中，动态调整第二批生产量

## 四、鸟巢限定款专属产品机会清单

> 以下是只有「鸟巢×艺人联名限定款」才能做的产品方向，独占性最强。
> 这些方向不受艺人自有文创分流影响，是鸟巢限定款的核心竞争优势。

### 类型1：含鸟巢官方元素的联名产品

- 带鸟巢官方LOGO/认证标的联名产品（需授权）
- 鸟巢坐标数据（经纬度39°59'35"N 116°23'26"E）
- 「在鸟巢演出」的场次日期/证书类产品
- 鸟巢建筑线稿/剪影与艺人符号的融合设计

**独占性**：⭐⭐⭐（艺人自有店永远无法合法提供这些元素）

### 类型2：本次演出专属叙事产品

- 含「首次/唯一/特殊里程碑」叙事的产品
- 含演出现场专属内容的产品（演出歌单/特殊彩蛋/现场互动梗）
- 演出现场专属编号产品（「第XXXXX号鸟巢见证者」）

**独占性**：⭐⭐⭐（时效性决定了永久无法复制）

### 类型3：鸟巢体量才能成立的产品

- 9万人同步应援道具（发光海/颜色矩阵）——小场馆无此效果
- 鸟巢夜晚户外的特殊体验类产品

**独占性**：⭐⭐（艺人自有店可以卖类似产品，但「鸟巢9万人的夜晚」叙事只属于这里）

## 五、保底品类清单（鸟巢限定款版本）

> 以下品类是演唱会场景下几乎零失败率的基础配置。
> **但每款必须有「鸟巢限定款独占理由」，否则降为普通周边，失去限定款定价优势。**

| 品类 | 鸟巢限定款独占理由 | 艺人自有产品分流风险 |
|-----|----------------|----------------|
| 票根保护套 | 含鸟巢经纬度+演出日期，内侧专属叙事文字 | 低（艺人自有通用款无法有鸟巢元素） |
| 贴纸套组 | 图案包含鸟巢×艺人融合设计+场次日期 | 低–中（需与艺人自有贴纸设计明显区分） |
| T恤（1–2款） | 含「鸟巢×演唱会」专属版式，非艺人通用款 | 中（须与艺人自有T恤形成明显差异） |
| 限量礼盒 | 鸟巢联名背书+叙事彩蛋+限量编号 | 低（艺人自有礼盒通常不含鸟巢联名元素） |

## 六、品类禁入清单（鸟巢限定款版本）

| 禁入方向 | 失败原因 |
|---------|---------|
| 与艺人自有文创「品类+价格」双重重叠的产品 | 直接分流，限定款没有差异化竞争力 |
| 没有鸟巢限定理由的通用设计产品 | 丧失「稀缺性溢价」，等于自降为普通周边 |
| 三维发现框架三问全无的大路货 | 无传播触发点，对粉丝无购买动机 |
| 不适合鸟巢物理条件的产品 | 现场体验差，传播效率低 |
| 定价虚高但感知价值不足的产品 | 粉丝感知「不值」，口碑负面 |
| 超出供应商风险承受度的生产量 | 滞销直接转化为无法变现的库存损失 |

## 七、历史爆款底层逻辑总结

| 底层逻辑 | 说明 | 鸟巢限定款如何应用 |
|---------|-----|----------------|
| 符号垄断 | 只有这个艺人才有这个符号 | 艺人核心符号+鸟巢专属元素双重垄断 |
| 集齐驱动 | 多款，粉丝想集齐 | 两场次各一款限定+联合款=三件套集齐逻辑 |
| 现场仪式 | 与「那个夜晚」绑定 | 鸟巢夜晚的应援道具，离开就买不到 |
| 叙事彩蛋 | 产品里藏着粉丝专属的故事 | 礼盒内信件/公民证/见证者编号 |
| 可晒性 | 天然适合拍照发社媒 | 开箱视频/穿戴上身/夜晚发光效果 |
| 唯一性证明 | 编号/限量，「我是那X个人之一」| 鸟巢限定款的核心价值主张 |
| 稀缺截止叙事（鸟巢限定款专属）| 「演出结束后永远买不到了」| 这是普通艺人自有产品无法提供的购买紧迫感 |
